Ubuntu Server Web Hosting Instructions

Introduction

This website provides step-by-step instructions to set up an Ubuntu Server for hosting web content locally. Additionally, the same website can be hosted remotely using Vercel, a third-party cloud hosting platform, allowing access from anywhere over the internet.

Use this guide to:

  • Install Ubuntu Server in VirtualBox
  • Configure Apache and serve HTML pages locally
  • Use GitHub to manage your project repository
  • Deploy your website to Vercel for remote access

1️⃣ Install Requirements

  • Oracle VM VirtualBox
  • Ubuntu Server (Download latest LTS ISO)

2️⃣ Create Ubuntu Server VM

  • Click New
  • Type: Linux
  • Version: Ubuntu (64-bit)
  • RAM: 2GB minimum
  • Disk: 20GB (VDI, Dynamically allocated)
  • Attach the ISO and start the VM

3️⃣ Install Ubuntu Server

  • Choose language
  • Keep network on DHCP
  • Create username + password
  • ✔ Install OpenSSH Server
  • Use entire disk
  • Finish installation and reboot
  • Login after reboot

4️⃣ Install Apache Web Server

  • Update system: sudo apt update & sudo apt upgrade -y
  • Install Apache: sudo apt install apache2 -y
  • Check status: sudo systemctl status apache2

5️⃣ Create Your HTML File

  • Default web folder: /var/www/html/
  • Edit default page: sudo nano /var/www/html/index.html
  • Example content: 
    <html>
<head>
<title>My Server</title>
</head>
<body>
<h1>Hello from Ubuntu Server!</h1>
</body>
</html>
  • Save and exit

6️⃣ Change Network to Bridged Adapter

  • Power off VM
  • Settings → Network → Adapter 1
  • Change from NAT ➜ Bridged Adapter
  • Choose your network card (WiFi/Ethernet)
  • Start VM again
  • VM now gets an IP from your router

7️⃣ Get Ubuntu Server IP Address

  • Run: ip a
  • Look for an IP like 192.168.1.105
  • This is your server IP

8️⃣ Allow Firewall

  • If firewall is active: sudo ufw allow 80
  • Enable firewall: sudo ufw enable

9️⃣ Access from Another PC

  • Open a browser on another PC in the same network
  • Type: http://192.168.1.105 (replace with your server IP)
  • You should see: Hello from Ubuntu Server!

GitHub Commands in Virtual Ubuntu Server CLI

🔹 Configure Git User

  • Set your name: git config --global user.name "Your Name"
  • Set your email: git config --global user.email "you@example.com"
  • Check configuration: git config --list

🔹 Initialize Repository & Commit

  • Navigate to your project folder (e.g., /var/www/html)
  • Initialize git: git init
  • Add files: git add .
  • Commit files: git commit -m "Initial commit"

🔹 Connect to GitHub & Push

  • Create a GitHub repository on your account (e.g., my-website).
  • Add remote: git remote add origin https://github.com/username/my-website.git
  • Push files: git push -u origin main

🔹 Pull Updates from GitHub

  • To fetch changes from GitHub: git pull origin main
  • Resolve conflicts if needed, then commit changes.

Vercel Web Hosting Instructions

🔹 Deploy via GitHub Repository

  • Go to Vercel and create a free account.
  • Click New Project → Import Git Repository.
  • Select your GitHub repository containing your website files.
  • Follow prompts for project name, scope, and build settings.
  • Vercel will automatically deploy your site from GitHub.
  • Your website will be live at a .vercel.app URL.
  • To redeploy after updates, just push changes to GitHub; Vercel auto-deploys.

🔹 Deploy via Vercel CLI (Optional)

  • Install Vercel CLI: npm install -g vercel
  • Login via CLI: vercel login
  • Navigate to project folder
  • Deploy: vercel --prod
  • Follow prompts; website is live at .vercel.app
